# BookMoa 기본 계산 로직 설명

이 문서는 현재 `src/App.jsx`에 구현된 견적 계산 로직을 쉽게 파악하기 위해 작성되었습니다.

## 1. 계산 엔진 위치

- 일반 인쇄 견적: `calcQuote(cfg, pricing)`
- 커스텀 상품 견적: `calcCustomQuote(prod, selections, quantity)`
- 위치: `src/App.jsx`

## 2. 계산에 쓰는 데이터(입력 원천)

견적 계산은 아래 2가지 데이터를 조합해 수행합니다.

- 사용자 입력 사양 `cfg`
  - 예: `format`, `pages`, `quantity`, `printType`, `innerPaper`, `innerSide`, `coverPaper`, `coverSide`, `coating`, `binding`, `endpaper`, `postProcessing`
- 가격 마스터 `pricing`
  - 기본값: `DEF_PRICING`
  - 관리자에서 수정/엑셀 임포트 후 저장된 값(`p4-pricing`)이 있으면 해당 값 사용

`DEF_PRICING` 주요 테이블:

- `printTypes`, `printTable`: 페이지 구간별 인쇄 단가
- `innerPapers`: 내지 종이/절수별 단가
- `sideRate`: 과거 방식 호환용(현재 `calcQuote`의 내지 단가 계산은 `innerPapers` 사용)
- `coverPrintRate`: 표지 단면/양면 인쇄 단가
- `coverPapers`: 표지 종이 단가
- `coatingTable`: 코팅 단가
- `bindingTypes`, `bindingTable`: 부수 구간별 제본 단가
- `endpapers`: 면지 단가
- `postProc`: 후가공 단가

## 3. 일반 인쇄 견적 계산식 (`calcQuote`)

### 3.1 항목별 계산

1. 인쇄비  
   - `printTable`에서 `pages`에 맞는 구간 행을 찾고(`lookupLE`), 선택한 `printType` 열의 단가를 가져옴  
   - 계산: `인쇄비 = 인쇄단가(pu) * pages`

2. 내지비  
   - `innerPapers[innerPaper][innerSize]`에서 1장 단가 조회  
   - `innerSize`는 판형 기준 매핑(`formatMap[format].innerSize`)  
   - 양면이면 페이지당 단가를 `leafCost / 2`로 계산  
   - 계산: `내지비 = (양면 ? leafCost/2 : leafCost) * pages`

3. 면지비  
   - `endpapers[endpaper][format]`  
   - 계산: `면지비 = 해당 단가`

4. 표지 종이비  
   - 기본적으로 `formatMap[format].coverSize` 사용  
   - 단, `무선날개/양장 + (A5/B5/A4)` 조합은 `3절`로 강제  
   - `coverPapers[coverPaper][coverSize]` 조회  
   - 계산: `표지종이비 = 해당 단가`

5. 표지 인쇄비  
   - `coverPrintRate[coverSide]`  
   - 계산: `표지인쇄비 = 해당 단가`

6. 코팅비  
   - 표지 절수(`coverSize`) 기준으로 `coatingTable[coverSize][coating]` 조회  
   - 계산: `코팅비 = 해당 단가`

7. 제본비  
   - `bindingTable`에서 `quantity`에 맞는 구간 행을 찾고(`lookupLE`), `binding` 열 단가 사용  
   - 계산: `제본비 = 해당 단가`

8. 후가공비  
   - `postProcessing` 배열의 항목들을 `postProc`에서 조회해 합산  
   - 계산: `후가공비 = 선택 항목 단가 합`

### 3.2 합계 계산

- 1부 단가 합계:  
  `up = 인쇄비 + 내지비 + 면지비 + 표지종이비 + 표지인쇄비 + 코팅비 + 제본비 + 후가공비`
- 단가 반올림:  
  `unitPrice = round(up)`
- 공급가(소계):  
  `subtotal = unitPrice * quantity`
- 부가세:  
  `vat = round(subtotal * 0.1)`
- 총합계:  
  `total = subtotal + vat`

## 4. 커스텀 상품 계산식 (`calcCustomQuote`)

커스텀 상품은 일반 인쇄 테이블이 아니라 상품 정의 데이터(`customProducts`)를 사용합니다.

1. 기본가 선택 (`qtyTiers`)
   - `quantity` 이상인 구간 중 마지막 구간의 `basePrice`를 기본가로 사용
2. 옵션 가감 (`optGroups -> choices -> priceAdj`)
   - 선택 옵션들의 `priceAdj`를 합산(플러스/마이너스 가능)
3. 1부 단가 계산
   - `up = max(0, 기본가 + 옵션가감합)`
4. 총액 계산
   - `unitPrice = round(up)`
   - `subtotal = unitPrice * quantity`
   - `vat = round(subtotal * 0.1)`
   - `total = subtotal + vat`

## 5. 현재 로직 검증 결과

### 5.1 내지 종이(`innerPaper`) 반영 여부

- 현재 `calcQuote`는 `innerPapers[innerPaper][innerSize]`를 직접 조회해 내지비를 계산합니다.
- 즉, `모조80`, `아트지150` 같은 종이 종류/평량 차이가 내지비에 반영됩니다.
- 양면 선택 시 페이지당 단가를 1/2로 계산합니다.

### 5.2 "내지 종이 반영 여부" 옵션화 상태

- 관리자 설정에 `useInnerPaperCost` 옵션이 추가되어 런타임에서 ON/OFF 제어가 가능합니다.
- ON (`true`, 기본값): `innerPapers` 기반 계산 사용
- OFF (`false`): 기존 `sideRate` 기반 계산 사용
- 주요 계산 경로(구성 화면 실시간 견적, 장바구니 수량 변경, 재주문, 홈 미리보기)에 옵션이 반영됩니다.

### 5.3 부가세율 설정값(`settings.taxRate`)과 계산식 불일치 가능성

- 화면에는 부가세율 설정(`settings.taxRate`)이 존재합니다.
- 하지만 계산 함수는 `0.1`(10%) 고정값을 사용합니다.
- 표기와 실제 계산이 달라질 가능성이 있습니다.

### 5.4 배송비는 총액 미포함(현재 정책)

- 설정에 `deliveryFee`가 있어도 견적 총액 계산에 포함하지 않습니다.
- 견적서 문구도 "배송비 별도"로 표기되어 현재 정책은 별도 청구로 보입니다.

## 6. 참고 코드 포인트

- `lookupLE(val, rows, key)`: 구간 테이블에서 `val` 이하 최대 행 선택
- `calcQuote(...)`: 일반 인쇄 견적 계산
- `calcCustomQuote(...)`: 커스텀 상품 견적 계산
- `DEF_PRICING`: 기본 가격 마스터
- `p4-pricing`: 저장된 가격 마스터 키
